# 敲きの技術

これまでに敲きとは何か，その際の心持ちは何なのかについて説明してきました．  
本稿ではそれらを実現する Google Document の技術面での話をします．[^1]

## 目標

- Google Document でコメント機能を用いた敲きができるようになる．

## 前提

- Google アカウントを持っている．

## コメントを付ける

### 1. コメントの範囲の選択

コメントを付けたい箇所を選択します．この時選択した範囲にわたってコメントはハイライトされます．

<p align="center">
  <image src="../assets/3_1-1.png" width="500px">
</p>

### 2. コメントボックスの生成

右にある緑と青のマークのうち，青色の方（中にプラスマークがある方）をクリックします．
すると，コメントを入力するボックスが出てきます．  
なお，Mac の人は `Command + Option + M`，Windows の人は `Ctrl + Alt + M` を押すことでもボックスが現れます．

<p align="center">
  <image src="../assets/3_1-2.png" width="500px">
</p>

### 3. コメントの送信

コメントを書き込んで，下の「コメント」を押すとコメントを送信することができます．  
なお，`Command + Enter`（Mac）/ `Ctrl + Enter`（Windows）でもコメントを送信することができます．

<p align="center">
  <image src="../assets/3_1-3.png" width="500px">
</p>

## コメントに反応する

技術面の話というよりは，どうするとやりやすいかというノウハウの話をします．

### 1. ドキュメントのコピーの作成

稿ごとに第１稿，第２稿，…… とドキュメントを分けることで以前の稿へのアクセシビリティが上がり，バージョン管理が容易になります．  
第 n 稿のドキュメントの「ファイル」→「コピーを作成」から第 n + 1 稿を作成しましょう．
以下，項目の先頭の【第 n 稿】【第 n + 1 稿】はその稿のドキュメントで作業を行うことを指します．

### 2.【第 n 稿】敲きへの返信

第 n 稿に付いているコメントへの返信を行います．第 n + 1 稿には，第 n 稿に付けられたコメントは反映されないためです．  

#### 2-1. 返信ボックスの生成

まず，返信するコメントをクリックします．この時，下に「@ を使用して返信またはユーザーを追加」と書かれたボックスが出てくるので，そこをクリックします．

<p align="center">
  <image src="../assets/3_2-1.png" width="500px">
</p>

#### 2-2. 返信

次に，返信内容を書き込んで下の「返信」を押すと返信することができます．  
なお，`Command + Enter`（Mac）/ `Ctrl + Enter`（Windows）でも返信することができます．

<p align="center">
  <image src="../assets/3_2-2.png" width="500px">
</p>

### 3.【第 n + 1 稿】敲きの反映

敲きを受諾し反映させる場合は，その内容を第 n + 1 稿に反映させます．

### 4.【第 n 稿】コメントの処理

敲きを受諾し，第 n + 1 稿に反映させた場合はそのコメントを「解決済み」にしましょう．
コメントの右上のチェックマークを押すことで「解決済み」にできます．

他方，敲きの内容を拒否した場合はそのコメントは残しておきましょう．
「解決済み」にするとスレッドに返信して議論を行うことがやや手間になります．敲いた側が疑問に思ったことはすぐに聞けるように，コメントはオープンにしておくと良いでしょう．

### 5. 第 n + 1 稿のリリース

上の [敲きの反映](#第-n--1-稿敲きの反映) と [コメントの処理](#第-n-稿コメントの処理) を第 n 稿に付けられた全ての敲きのコメント[^2]に繰り返します．それらが完了すると，全ての敲きのコメントが

- 解決済みになっている
- スレッドに返信した形で残っている

のいずれかを満たす状態になります．  
そうなったら第 n + 1 稿をリリースしましょう．お疲れ様でした！

## 練習

1. 今，班や参加企画で敲きを求められている Google Document を用いた成果物（全体／コンテンツ企画書，参加要項，ファシリマニュアル，ワークシートなど）があれば，一通り見た上で，必要であれば敲きを入れてみましょう．

2. 今，自分が敲きを募集している成果物があれば，それに敲きが来ていないか確認しましょう．来ている場合はそれらを反映させた第 n + 1 稿を作成してみましょう．
